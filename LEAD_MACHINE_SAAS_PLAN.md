# Lead Machine SaaS Plan

## Vision

Lead Machine should become the go-to seller desk for many seller types. A seller logs in, chooses what they sell, searches or selects a market, and the system finds leads plus the right context for that sales motion.

## Core Loop

1. Login/team workspace.
2. Seller setup: what they sell, geography, ideal customer, disqualifiers.
3. Lead discovery: Brreg identity plus public presence/contact data.
4. Source Fusion: Proof & Confidence across identity, contact, location and seller fit.
5. Lead desk: explicit seller work queues, lead cards, notes, outcomes, follow-ups.
6. Selected-lead enrichment when more context is worth it.
7. Email/calendar/CRM connections after the manual workflow is proven.

## Keep Out Of Main App For Now

- Demo generator.
- Landing-page test project.
- Website-redesign-specific language.
- AI outreach/call scripts.
- Proff as a dependency.
- SSB market panels.
- Complex multi-agent orchestration.
- Dashboard polish before workflow quality.
- Broad scraping.
- Historical monitoring.
- Parallel workers.

## Current Beta

The local app and Netlify friend beta prove the seller desk loop: find leads, read Proof & Confidence, work explicit queues, call manually, log notes, set follow-ups, and export. This is the correct base before adding login, email, CRM, paid providers, or richer background agents.

## Background Agent Direction

A background agent should help with narrow product jobs:

- refresh saved market searches,
- detect stale/changed lead data,
- suggest which leads need verification,
- prepare follow-up queues,
- summarize workspace health.

It should not sell for the user, scrape private profiles, generate pitch scripts, call, email, or override seller judgment.

## Data Source Direction

Source Fusion is the foundation before 1881 or Proff. 1881 should later improve contact/directory confidence if terms allow the SaaS use case. Proff should later improve commercial/economy confidence only after Brreg confirms org.nr. Eniro/Gule Sider, SSB, scraping and broad OSINT stay out until the seller workflow and source-confidence model are proven.
