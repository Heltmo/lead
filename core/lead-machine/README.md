# Lead Machine

Single-command runner for the Webconsult lead intelligence flow.

It coordinates existing modules:

```text
discovery -> orchestrator audit -> lead-pack-runner
```

It does not implement scoring, outreach, Proff, frontend, saved searches, or sales scripts.

## Usage

```bash
cd core/lead-machine
npm run lead-machine -- \
  --query "advokater i Gol" \
  --provider google-places \
  --max-results 5 \
  --search-scope strict \
  --enrich-company-profile false
```

## Search Scope

- `strict`: exact-location only, default
- `nearby`: reserved for future radius behavior
- `regional`: explicit fallback; non-exact candidates are marked `regional_fallback`

## Outputs

```text
core/lead-machine/runs/<run-id>/
  discovery/
    lead-candidates.json
    discovery-summary.json
    handoff.jsonl
  lead-packs/
    lead-packs.json
    lead-packs.csv
    summary.json
  lead-machine-summary.json
```

The summary includes:

- query
- provider
- searchScope
- maxResults
- sourceRunPath
- leadPackOutputPath
- totalDiscovered
- totalIncluded
- locationQualityCounts
- callPriorityCounts
- lowSupply
- fallbackAvailable
- fallbackUsed
- recommendedExpansion
- companyProfileEnabled
- companyProfileCounts
- economyStatus

## Product Boundary

The machine owns discovery, enrichment, ranking, evidence, caution, and export.

The seller owns angle, wording, outreach, timing, relationship, and closing.

No prepared sales scripts are generated by default.
