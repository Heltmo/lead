# Lead Machine

Single-command runner for the Webconsult lead intelligence flow.

It coordinates existing modules:

```text
discovery -> orchestrator audit -> lead-pack-runner
```

It does not implement scoring, outreach, Proff, frontend, saved searches, or sales scripts.

## Usage

```bash
cd core/lead-machine
npm run lead-machine -- \
  --query "advokater i Gol" \
  --provider google-places \
  --max-results 5 \
  --search-scope strict \
  --enrich-company-profile false
```

## Search Scope

- `strict`: exact-location only, default
- `nearby`: reserved for future radius behavior
- `regional`: explicit fallback; non-exact candidates are marked `regional_fallback`

## Outputs

```text
core/lead-machine/runs/<run-id>/
  discovery/
    lead-candidates.json
    discovery-summary.json
    handoff.jsonl
  lead-packs/
    lead-packs.json
    lead-packs.csv
    summary.json
  lead-machine-summary.json
```

The terminal output and `lead-machine-summary.json` are meant to be operator-readable. They include:

- query
- provider
- searchScope
- maxResults
- outputDir
- sourceRunPath
- leadPackOutputPath
- totalDiscovered
- includedLeadCount
- totalExcludedByLocation
- locationQualityCounts
- callPriorityCounts
- lowSupply
- fallbackAvailable
- fallbackUsed
- recommendedExpansion
- companyProfileEnabled
- companyProfileCounts
- economyStatus
- nextRecommendedAction


## Interpreting Results

- `strict` with `includedLeadCount=0` and `fallbackAvailable=true`: run again with `--search-scope nearby` or `regional`.
- `strict` with `lowSupply=true`: exact-location supply is limited; review exact leads first or expand if volume matters.
- `regional` with `fallbackUsed=true`: fallback leads are not local leads. Review location warnings before using them.
- Results with HIGH leads: review HIGH first.
- Results with no HIGH but MEDIUM leads: use MEDIUM as the shortlist layer.

## Product Boundary

The machine owns discovery, enrichment, ranking, evidence, caution, and export.

The seller owns angle, wording, outreach, timing, relationship, and closing.

No prepared sales scripts are generated by default.

## Run Modes

`--mode fast` creates lead packs directly from discovery output and skips full website audit. It is intended for quick seller scans and marks leads as `verify` until Deep mode is run.

`--mode deep` is the default full pipeline: discovery, website audit, business signals, opportunity compression, commercial pressure, and lead packs.

Example:

```bash
npm run lead-machine -- --query "advokater i Moss" --provider google-places --max-results 20 --search-scope strict --mode fast
```
